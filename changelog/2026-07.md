# Changelog: July 2026 (v2026.07)

We have released the initial stable version of the `moul` console and `moul-dev` engine. The project provides a single, self-contained toolkit that handles dynamic database schemas, background queue workers, and user sessions without the operational overhead of a distributed stack.

Modern web architectures often require running separate servers for databases, job queues, key-value stores, and session tracking. This configuration drains developer energy and increases hosting costs. We believe a single, compiled Go binary running on a modest server can easily handle all of these responsibilities.

## Core Capabilities

### 1. Dynamic Mouls and Data Rules
At runtime, you can define, update, or remove database tables—called Mouls—via a REST API. 
- **Flexible Schema Types**: Supports standard databases (`base`), authentication targets (`auth`), background queue tables (`worker`), and session tracking tables (`analytic`).
- **Dynamic Rule Engine**: Restricts read/write access using a custom translation syntax (e.g. `auth.id == author_id`). Rules are parsed dynamically with the `expr` compiler.

### 2. SQLite-Backed Background Workers
Asynchronous tasks are managed using a queue structure inspired by Elixir's Oban. 
- **Queues and Priorities**: Jobs are stored directly in your SQLite database. You can configure execution priorities, define queue limits, and schedule runs.
- **Robust Failures**: Automatically retries failed jobs using an exponential backoff strategy, avoiding thread-blocking bottlenecks.

### 3.Observability and Session Analytics
observability is integrated directly into the HTTP router middleware[^1].
- **Observability Tables**: The engine logs HTTP requests asynchronously into system tables, avoiding write bottlenecks on incoming requests.
- **Session Resolvers**: Automatically tracks user visits and maps campaign UTM parameters, device types, and geographical regions.

[^1]: Telemetry uses MaxMind GeoIP2 resolution and parses cookies or custom headers to resolve user sessions without impacting request latency.

### 4. The Charm Bubble Tea TUI
We built a terminal user interface (`moul`) to manage the engine without leaving your command line.
- **Realtime Control**: Connects securely to the engine to display background job queues, active database tables, and request logs.
- **System Configurator**: Allows configuring S3 file uploads, Litestream replication settings, rate limiters, and allowed IP ranges.
