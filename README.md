# Moul Dynamic API Engine

`moul-dev` is a lightweight, self-contained dynamic database and authentication engine in Go, inspired by PocketBase.

---

## Key Features

1. **Dynamic Mouls (Tables)**: Create, list, and delete database tables and schemas at runtime via HTTP API.
2. **Dynamic Record CRUD**: Perform complete CRUD operations on any dynamic moul using raw JSON payloads.
3. **Bcrypt Password Hashing**: Auth-type mouls automatically hash passwords securely when inserting or updating records.
4. **JWT-Based Authentication**: Issue signed JWT tokens on successful logins and parse/verify them automatically using Bearer token middleware.
5. **Rule Authorization Engine**: Enforce HCL-like rules (e.g. `auth.id == author_id` or `auth.id != nil`) dynamically using the fast `expr-lang/expr` library.
6. **Single Binary SQLite**: Driven by `github.com/pocketbase/dbx` and the CGO-free `modernc.org/sqlite` driver for lightweight, zero-configuration local deployment.

---

## Technical Stack

- **HTTP Framework**: [Echo v4](https://echo.labstack.com)
- **Database Abstraction**: [pocketbase/dbx](https://github.com/pocketbase/dbx)
- **SQLite Driver**: [modernc.org/sqlite](https://modernc.org/sqlite) (Pure Go, CGO-free)
- **Expression Engine**: [expr-lang/expr](https://github.com/expr-lang/expr)
- **JWT Handling**: [golang-jwt/jwt](https://github.com/golang-jwt/jwt)

## Getting Started

### Prerequisites
- **Go**: Version 1.25 or higher

### Run Local Server
To start the Echo HTTP server on port `:8090` (this automatically creates or opens the local SQLite database file `moul-local.db`):
```bash
make run
```

### Run Go Integration Tests
To execute the comprehensive unit and integration tests (which run against a fast in-memory SQLite database):
```bash
make test-go
```

## API Documentation

### 1. Moul (Schema) Management

#### Create a Moul
- **HTTP Method**: `POST`
- **Path**: `/api/mouls`
- **Request Body**:
```json
{
  "name": "posts",
  "type": "base",
  "fields": [
    { "name": "title", "type": "text" },
    { "name": "body", "type": "text" },
    { "name": "author_id", "type": "text" }
  ],
  "rules": {
    "listRule": "",
    "viewRule": "",
    "createRule": "auth.id != nil",
    "updateRule": "auth.id == author_id",
    "deleteRule": "auth.id == author_id"
  }
}
```

#### List Mouls
- **HTTP Method**: `GET`
- **Path**: `/api/mouls`

#### Delete a Moul
- **HTTP Method**: `DELETE`
- **Path**: `/api/mouls/:name`

---

### 2. Record (Data) Management

All record IDs generated by the engine are prefixed with the singular name of the moul (e.g. `user-LpI7pWl7NnOq62K` or `post-bSvPNdw4Y1TwXQp`).

#### Create a Record
- **HTTP Method**: `POST`
- **Path**: `/api/mouls/:moulName/records`
- **Request Body**:
```json
{
  "title": "My Post Title",
  "body": "Hello world",
  "author_id": "user-LpI7pWl7NnOq62K"
}
```
*(If the moul is an `"auth"` type, requires `username`, `email`, `password`, and `passwordConfirm` fields).*

#### Authenticate with Password (Auth Moul only)
- **HTTP Method**: `POST`
- **Path**: `/api/mouls/:moulName/auth-with-password`
- **Request Body**:
```json
{
  "identity": "usera@example.com", // email or username
  "password": "password123"
}
```
- **Response**: Returns a JSON object with `"token"` (JWT) and `"record"` info.

#### List Records
- **HTTP Method**: `GET`
- **Path**: `/api/mouls/:moulName/records`
*(Automatically filters out records that do not pass the moul's `listRule` for the current user).*

#### Get a Record
- **HTTP Method**: `GET`
- **Path**: `/api/mouls/:moulName/records/:id`

#### Update a Record
- **HTTP Method**: `PATCH`
- **Path**: `/api/mouls/:moulName/records/:id`

#### Delete a Record
- **HTTP Method**: `DELETE`
- **Path**: `/api/mouls/:moulName/records/:id`

## Manual Flow Verification (cURL Script)

To manually test and watch the server process requests (creating schemas, registering accounts, validating passwords, fetching records, verifying authorization, and cleaning up), you can run:

```bash
make test-flow
```
*(Ensure the server is running via `make run` in a separate terminal before executing this command).*
